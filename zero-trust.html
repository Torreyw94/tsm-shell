HOW-TO GUIDES
Step-by-step operational guides for TSM Zero-Trust platform. Click any guide to expand.
🔐 How to Onboard a New User▶ Expand
Create, provision, and enforce MFA for new team members
01
Navigate to Users Tab
Click "Users" in the top navigation bar. You'll see the full user directory with filters for role, status, and risk level.
02
Click "+ Create User"
Fill in Full Name, Email (@tsmatter.com domain auto-assigns Standard User role), Department, and select the appropriate Role from the dropdown.
03
Configure MFA Requirement
Ensure "Require MFA on first login" is checked. This is mandatory for all roles except Guest. The user will be prompted to enroll during first sign-in.
04
Send Onboarding Email
Check "Send onboarding email" to automatically dispatch credentials and setup instructions. Click Create User — the new account appears as "Pending" status until first login.
05
Verify in Audit Log
Go to Audit Log tab and filter by Event Type: ADMIN to confirm the user creation was logged with proper chain-of-custody.
🚨 How to Respond to a Threat Alert▶ Expand
Investigate, contain, and document security incidents
01
Identify the Alert
Alerts appear as red banners on the Dashboard and in the Live Threat Feed panel. Critical events are also listed in the Threat Intel tab with MITRE ATT&CK mappings.
02
Run AI Investigation
Click "Investigate" next to the threat event, or type your query in the Neural Processor. Use the Incident Response presets for guided analysis including TTP correlation.
03
Contain the Threat
For suspicious users: go to Users tab → find the UID → click "Sus." to suspend the session immediately. For IP-based threats: use "Block IPs" in the Threat Intel tab.
04
Use Quick Actions
The Threat Feed panel has four quick-action buttons: Security Audit, Lockdown, IR Report, and Access Review — each fires a targeted AI analysis prompt.
05
Document the Incident
Use the "Post-Incident Review" preset in the Incident Response sector to generate a PIR template. Export from Audit Log with Evidence Package for regulatory records.
⚖️ How to Use Legal Contract Analysis▶ Expand
Analyze contracts, NDAs, and vendor agreements for risk
01
Open the Neural Processor
The AI panel is at the top of the Dashboard. Expand the "Legal & Contract Risk" sector by clicking its header — you'll see 7 analysis presets.
02
Select the Right Preset
Choose from: Contract Risk Analyzer, NDA Enforceability Review, IP Assignment Gap Analysis, Vendor SLA Compliance Review, Litigation Hold Check, Wrongful Termination Audit, or Regulatory Filing Audit.
03
Paste Your Contract Text
After clicking a preset, the prompt loads into the input field. Append your contract language after the prompt text, then click ANALYZE. The AI will identify specific risk clauses and remediation language.
04
Review the Analysis
Results appear in the message panel above the input. The AI identifies indemnification risks, liability caps, jurisdiction issues, and provides specific amendment recommendations.
05
Escalate if Needed
For critical findings, copy the AI analysis and route to your legal team. All queries are logged in the Audit Trail under COMPLIANCE event type for record-keeping.
⚙️ How to Use Dev/Ops Cloud Analysis▶ Expand
Audit SaaS agreements, cloud costs, and developer contracts
01
Open Dev/Ops Sector
In the Neural Processor, click "Dev / Ops & Cloud" to expand the sector. Seven specialized presets cover the full engineering and cloud contract lifecycle.
02
Cloud FinOps Analysis
Use "Cloud FinOps Cost Allocation Analysis" to paste your AWS/Azure/GCP cost export. The AI identifies reserved instance gaps, showback model errors, and optimization opportunities with dollar estimates.
03
SaaS Agreement Review
Use "SaaS MSA Data Processing Review" for any SaaS vendor MSA. Paste the DPA section and the AI checks GDPR sub-processor disclosure completeness and breach notification SLAs.
04
SOC 2 Gap Review
Paste your vendor's SOC 2 Type II report text into "SOC 2 Type II Gap Review" to check complementary user entity controls and subservice carve-out risk before signing a vendor agreement.
05
Bug Bounty & API Terms
Use the Bug Bounty Audit preset to validate your program's safe harbor provisions, and the API Terms preset to check data portability obligations before publishing developer documentation.
👤 How to Offboard a Departing Employee▶ Expand
Safely revoke all access within the required SLA
01
Check Offboarding Queue
Go to Users tab. The Offboarding Queue panel on the right lists all terminated employees with remaining active access. Red banner indicates immediate action is required.
02
Revoke Active Sessions
Click "Revoke" next to each terminated user. This terminates all active sessions, invalidates JWT tokens, and removes access from all integrated systems including GitHub, PAM vault, and SaaS tools.
03
Suspend the Account
In the user table, find the user and click "Sus." to set status to Suspended. This prevents any new logins while preserving the account record for audit purposes.
04
Review Data Access History
Go to Audit Log and search by the user's UID. Review all ACCESS events from the last 30 days for any unusual data exports or system access that may indicate IP theft risk.
05
Confirm in Audit Trail
Filter Audit Log by Event Type: ADMIN and verify the revocation events are logged with timestamp, actor (your UID), and result: Success. This is required for SOC-2 CC6 control evidence.
📋 How to Run a Compliance Report▶ Expand
Generate SOC-2, HIPAA, GDPR, and NIST evidence packages
01
Check Current Posture
Navigate to the Compliance tab for a real-time view of SOC-2, GDPR/CCPA, and NIST CSF maturity. Any "Gap" items require remediation before generating an evidence package.
02
Use AI Compliance Presets
In the Neural Processor, expand "Compliance & Regulatory" and select the relevant framework: SOC-2 Evidence Package, HIPAA Gap Analysis, IRS-1075 Review, GDPR Audit, CCPA Report, or NIST CSF Assessment.
03
Generate Evidence Package
Go to the Audit Log tab and click "Evidence Package." This compiles WORM-locked log entries, control documentation, and access records into a formatted package for auditors.
04
Export to SIEM
Use "Export SIEM" to generate a structured log file compatible with Splunk, QRadar, or Sentinel. This provides the 90-day retention evidence required under HIPAA §164.312b.
05
Schedule Regular Reviews
The platform auto-triggers quarterly access certification campaigns. Use the "Access Certification Report" AI preset before each campaign to identify accounts needing review before the auditor does.
🔑 How to Manage RBAC Roles▶ Expand
Configure roles, permissions, and automated access policies
01
Review Existing Roles
Navigate to the RBAC tab. Six built-in roles are displayed with current user counts and permission sets: System Admin, Security Admin, Auditor, Power User, Standard User, and Guest.
02
Edit Role Permissions
Click "Edit Permissions" on any role card to open the role editor. Follow least-privilege principles — assign only the minimum permissions required for the role's function.
03
Create Custom Roles
Click "+ Create Custom Role" for department-specific access needs. Custom roles inherit from the Standard User baseline and layer specific permissions on top.
04
Configure Automated Policies
The bottom panel shows eight automated RBAC actions including session timeout, geo-blocking, and AI risk-based step-up auth. These apply globally to all roles — toggle via the policy editor.
05
Run Role Mining Analysis
Use the "Role Mining & Consolidation" AI preset periodically to identify redundant roles and users with excessive permissions. This is required evidence for SOC-2 CC6 quarterly access reviews.
🧠 How to Use the Neural Processor▶ Expand
Get the most from the AI security analysis engine
01
Use Sector Presets
Click any sector header (Identity, Threat, Compliance, Network, Incident Response, Legal, Dev/Ops) to expand its preset library. Presets are color-coded: red for security, cyan for legal, purple for DevOps.
02
Chain Prompts Together
After an AI response, type follow-up questions in the input field. The Neural Processor maintains context within the session — use this to drill down from high-level risk summaries to specific remediation steps.
03
Paste External Data
For legal and DevOps analysis, paste contract text, cost reports, or SOC 2 report excerpts directly after the preset prompt. The AI will analyze the pasted content in context.
04
Use Quick-Action Buttons
Four quick-action buttons at the bottom of the Threat Feed (Security Audit, Lockdown, IR Report, Access Review) fire pre-configured AI queries optimized for those specific scenarios.
05
Interpret Results
AI responses are limited to 2-4 sentences for operational clarity. For deeper analysis, include specific UID numbers, timeframes, or clause text in your prompt. The system operates in an enterprise security context at all times.
