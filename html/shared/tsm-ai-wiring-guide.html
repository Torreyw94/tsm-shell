TSMatter AI Engine · Wiring Guide

How to connect tsm-ai-engine.js to every app in the hub.

Step 1 — Place the engine on your server

# Upload to your server once. Every app references this single file.
/html/shared/tsm-ai-engine.js

That's the only file you need to host. All apps reference it via a single script tag. No npm, no build step, no CDN calls.

Step 2 — Add one script tag to each app

<script src="/html/shared/tsm-ai-engine.js"></script>

Add this before your closing </body> tag in every HTML file in the hub.

Step 3 — Initialize for the right company + node

Call TSMai.init(profileId, nodeId) at the top of each app's script block. Pick the profile and node from the tables below.

<script>
  // HonorHealth · HC-Billing node
  TSMai.init('honorhealth', 'hc-billing');

  // Banner Health · HC-Strategist node
  TSMai.init('banner', 'hc-strategist');

  // Desert Financial · compliance node
  TSMai.init('desertfinancial', 'compliance');

  // Federation / Sovereign Strategist
  TSMai.init('federation', 'strategist');
</script>

Core methods — quick reference

TSMai.init()
Initialize with a company profile and node ID. Call once per app load.
TSMai.init('honorhealth', 'hc-billing')
TSMai.stream()
Send a prompt, stream result into an element with typewriter effect.
TSMai.stream(prompt, document.getElementById('out'), { thinkEl: thinkBar })
TSMai.pack()
Run a named pull pack. IDs: revenue, variance, denial, auth, compliance, executive, payer.
TSMai.pack('revenue', outEl)
TSMai.strategist()
Run a strategist analysis mode. Modes: priority, denial, revenue, compliance, payer, coding, forecast, board.
TSMai.strategist('denial', contextText, outEl)
TSMai.draft()
Draft a document. Types: appeal, auth, query, compliance, education, executive.
TSMai.draft('appeal', claimContext, outEl)
TSMai.chat()
Multi-turn chat — maintains conversation history automatically.
TSMai.chat(userMsg, chatContainer)
TSMai.score()
Calculate a 0-100 health score from live data. Colors the element automatically.
TSMai.score(data, scoreEl, labelEl)
TSMai.signals()
Generate and render live strategic signals into a container.
TSMai.signals(signalContainer)
TSMai.wire()
Attach AI logic to existing DOM buttons/inputs without rewriting HTML.
See the wiring pattern below.
TSMai.wireAll()
Auto-scan the page for data-tsmai attributes and wire everything. Zero-JS option.

Wiring patterns — 3 ways to wire any existing app

Pattern A — TSMai.wire() — attach to existing buttons, zero HTML changes

Best for: apps that already have buttons + output divs. Just attach the logic.

TSMai.init('honorhealth', 'hc-billing');

// Wire a pull pack button to an output div
TSMai.wire({
  trigger:  '#btn-revenue-pack',    // existing button
  output:   '#revenue-out',          // existing output div
  thinking: '#think-bar',            // optional loading bar
  error:    '#err-out',              // optional error div
  type:     'pack',
  packId:   'revenue'
});

// Wire a strategist button
TSMai.wire({
  trigger:  '#btn-run-strat',
  output:   '#strat-out',
  input:    '#strat-context',   // reads value from this input
  thinking: '#think-bar',
  type:     'strategist',
  mode:     'denial'
});

// Wire a chat send button
TSMai.wire({
  trigger:  '#chat-send-btn',
  output:   '#chat-messages',
  input:    '#chat-input',
  type:     'chat'
});

// Wire a document drafter
TSMai.wire({
  trigger:  '#btn-draft',
  output:   '#draft-out',
  input:    '#draft-context',
  type:     'draft',
  docType:  'appeal'
});

// Wire a health score
TSMai.wire({
  trigger:  '#btn-score',
  output:   '#score-dummy',   // required by wire(), can be any div
  type:     'score',
  scoreEl:  '#score-number',
  labelEl:  '#score-label'
});

// Wire live signals
TSMai.wire({
  trigger:  '#btn-refresh-signals',
  output:   '#signals-container',
  type:     'signals'
});

// Chained — wire multiple in one call sequence
TSMai
  .wire({ trigger:'#btn-a', output:'#out-a', type:'pack', packId:'denial' })
  .wire({ trigger:'#btn-b', output:'#out-b', type:'pack', packId:'auth' })
  .wire({ trigger:'#btn-c', output:'#out-c', type:'strategist', mode:'board' });

Pattern B — data-tsmai attributes — zero JavaScript required

Best for: rapid wiring. Add attributes to existing HTML elements, call TSMai.wireAll() once.

<body data-tsmai-profile="honorhealth" data-tsmai-node="hc-billing">

  <!-- Pull pack button -->
  <button
    data-tsmai="pack"
    data-tsmai-pack="revenue"
    data-tsmai-output="#revenue-out"
    data-tsmai-thinking="#think-bar"
  >Pull Revenue Pack</button>

  <!-- Strategist button -->
  <button
    data-tsmai="strategist"
    data-tsmai-mode="denial"
    data-tsmai-output="#strat-out"
    data-tsmai-input="#context-input"
  >Run Denial Analysis</button>

  <!-- Static prompt button -->
  <button
    data-tsmai="stream"
    data-tsmai-prompt="What is our top denial risk today?"
    data-tsmai-output="#quick-out"
  >Top Denial Risk</button>

  <!-- Score button -->
  <button
    data-tsmai="score"
    data-tsmai-score="#score-el"
    data-tsmai-label="#label-el"
    data-tsmai-output="#dummy"
  >Calculate Score</button>

</body>

<!-- Auto-wires everything on DOMContentLoaded -->
<script src="/html/shared/tsm-ai-engine.js"></script>

Pattern C — direct calls — full programmatic control

Best for: custom logic, conditional flows, or calling AI from your own event handlers.

TSMai.init('honorhealth', 'hc-strategist');

// Direct stream call
async function runMyAnalysis() {
  const out = document.getElementById('my-output');
  const text = await TSMai.stream(
    'What is the top revenue risk at HonorHealth right now?',
    out,
    { thinkEl: document.getElementById('my-loader') }
  );
  console.log('AI said:', text);
}

// Raw API call (returns text string, no UI)
const answer = await TSMai.call('Summarize the denial situation in 2 sentences.');

// Pull a pack, handle the result yourself
const packResult = await TSMai.pack('denial', outEl);

// Score with live data override
TSMai.updateData({ denialRate: 16.2, revenueAtRisk: 32000 });
await TSMai.score(null, scoreEl, labelEl);

// Chat with history
await TSMai.chat('What should I do about the Mesa denial spike?', chatContainer);
await TSMai.chat('And what should the Mesa OM do first?', chatContainer); // remembers context
TSMai.chatReset(); // clear history

Exact wiring for each app in the hub

honor-portal/nu-index.html

// In the existing <script> block, replace the callClaude() function body:
TSMai.init('honorhealth', 'hc-command');

// Replace runStrategist() with:
async function runStrategist() {
  const mode = document.getElementById('strat-mode').value;
  const ctx  = document.getElementById('strat-context').value;
  await TSMai.strategist(mode, ctx,
    document.getElementById('strat-out'),
    { thinkEl: document.getElementById('strat-thinking'),
      errEl:   document.getElementById('strat-err') }
  );
}

// Replace runPack() with:
async function runPack(packId) {
  await TSMai.pack(packId,
    document.getElementById('strat-out'),
    { thinkEl: document.getElementById('strat-thinking') }
  );
}

// Replace sendChat() with:
async function sendChat(preset) {
  const inp = document.getElementById('chat-inp');
  const msg = preset || inp.value.trim();
  if (!msg) return;
  inp.value = '';
  TSMai.ui.chatMessage(document.getElementById('chat-msgs'), msg, 'user');
  await TSMai.chat(msg, document.getElementById('chat-msgs'));
}

// Auto-score on load:
TSMai.score(null,
  document.getElementById('strat-score'),
  document.getElementById('strat-score-sub')
);

data-tsmai attribute reference

data-tsmai
Type: pack | stream | strategist | draft | chat | score | signals
data-tsmai-output
CSS selector for the output element. Required.
data-tsmai-thinking
CSS selector for the loading/thinking bar element.
data-tsmai-error
CSS selector for the error display element.
data-tsmai-input
CSS selector for the input/textarea to read the prompt from.
data-tsmai-pack
Pack ID. Used when data-tsmai="pack".
data-tsmai-mode
Strategist mode. Used when data-tsmai="strategist".
data-tsmai-prompt
Static prompt string. Used when data-tsmai="stream".
data-tsmai-doc
Document type. Used when data-tsmai="draft".
data-tsmai-score
CSS selector for the score number element.
data-tsmai-label
CSS selector for the score label element.
data-tsmai-profile (on body)
Auto-init profile ID on DOMContentLoaded.
data-tsmai-node (on body)
Auto-init node ID on DOMContentLoaded.

Adding a new company profile

When you onboard Banner, Dignity Health, or any new client — add their profile to the PROFILES object in tsm-ai-engine.js:

// Inside tsm-ai-engine.js — PROFILES object
dignityheath: {
  id: 'dignityhealth',
  company: 'Dignity Health',
  sector: 'healthcare',
  persona: 'VP Revenue Cycle',
  allowedNodes: ['hc-billing','hc-compliance','hc-insurance','hc-strategist'],
  disallowedDomains: ['mortgage','construction','schools'],
  offices: ['site-a','site-b','site-c','site-d'],
  priorities: ['network_denial_rate','site_variance','payer_mix'],
  currentData: { denialRate: 17.9, revenueAtRisk: 340000 },
  systemPrompt() {
    return `You are HC-Strategist for Dignity Health network.
Persona: ${this.persona}.
Focus on network-level analysis across all sites.
Prioritize: site variance, network denial rate, payer mix optimization.
Return: top issue, dollar impact, 3 actions, which site director acts first.`;
  }
}
